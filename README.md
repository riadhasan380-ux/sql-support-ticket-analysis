# SQL Support Ticket Analysis Project

## 📊 Project Overview
This project analyzes customer support ticket data using SQL to evaluate operational performance. The goal is to understand response times, SLA violations, ticket volume, and agent workload.

---

## 🎯 Business Objective
The objective of this project is to answer key business questions:
- How fast are support teams responding to tickets?
- Which tickets violate SLA (Service Level Agreements)?
- Which agents handle the most workload?
- Which issue categories generate the most tickets?

---

## 🗄️ Database Design
The database includes the following tables:
- **customers** → stores customer information
- **agents** → stores support agent details
- **categories** → defines issue types and SLA targets
- **tickets** → stores ticket activity and performance data

---

## 🛠️ Skills Demonstrated
- SQL Joins
- GROUP BY & Aggregations
- Date & Time Functions
- Business Data Analysis
- Relational Database Design

---

## 📈 Key Insights
- Some tickets exceeded SLA response time, indicating delays in support
- Certain categories (e.g., billing, login issues) generate higher ticket volume
- Agent workload is uneven, suggesting opportunities for better distribution
- SQL can be used to identify performance bottlenecks in operations

---

## 💡 Business Impact
This project demonstrates how SQL can help organizations:
- Improve customer support efficiency
- Monitor SLA compliance
- Optimize team performance
- Make data-driven decisions
